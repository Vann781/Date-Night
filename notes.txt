Project: Date-Night AI Agent
=============================
An AI-powered date night planner: describes a vibe → gets restaurant recommendations + backup plan + talking points.

BACKEND LIBRARIES
─────────────────
Flask            → Web server, receives POST requests from Flutter
Flask-CORS       → Allows cross-origin requests (Flutter → Render)
Gunicorn         → Production WSGI server (used on Render)
google-genai     → Google Gemini API client (model: gemini-3.1-flash-lite)
python-dotenv    → Loads .env file (API keys)
requests         → HTTP client (reserved for future external API calls)

HOW THE AI AGENT WORKS (Pipeline)
─────────────────────────────────
1. Flutter sends {vibe, cuisine, budget, location} to POST /api/plan-date
2. app.py validates the request via DatePlanRequest dataclass
3. orchestrator.py calls:
   a) Gemini → generates 5-6 real restaurants matching the criteria
   b) Gemini → analyzes each restaurant's vibe match, ranks them
   c) Gemini → generates a talking point for the #1 pick
   d) Gemini → writes a 1-2 sentence summary of the date plan
4. parser.py extracts structured JSON from each Gemini response
5. build_response() assembles {primary_pick, backup_pick, talking_point, summary}
6. Flask returns JSON → Flutter displays styled result cards

FRONTEND LIBRARIES
──────────────────
Flutter SDK      → Android app framework
http             → API calls to backend

DEPLOYMENT (Render)
───────────────────
- Build: pip install -r backend/requirements.txt
- Start: gunicorn backend.app:app
- Set env var: GEMINI_API_KEY in Render Dashboard
- Flutter points to: https://date-night-ky1n.onrender.com/api

KEY DESIGN DECISIONS
────────────────────
- No DuckDuckGo/Google Places API → Gemini generates everything from training data
- No pydantic → stdlib dataclasses (avoids Rust compilation on Render)
- Fallback: if Gemini API key has no quota → returns 6 curated Indian restaurants
- Glowing heart animation during loading (pure Flutter, no extra packages)
